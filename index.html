Доступность

Доступность — возможность использования интерфейса всеми, независимо от физических или технических ограничений

https://github.com/web-standards-ru/dictionary

Мотивация

  • Финансовая
  • Юридическая

Требования и рекомендации

  • ГОСТ Р 52872-2012
  • WCAG 2.0
  • Законодательные акты различных стран

Основные советы

  • Сохраняйте логическую структуру на разных страницах сайта
  • Если существует встроенный аналог - используйте его

Встроенные элементы

<div>Кнопка</div>
<span>Ссылка</span>

<button>Кнопка</button>
<a href="somepage.html">Ссылка</a>

Встроенные элементы

  • Получают фокус
  • Работают с клавиатурой
  • Имеют правильные роли
  • Другое: disabled, :link

Картинки

  • Делайте картинки с помощью тега img
  • Добавляйте описание с помощью атрибута alt

Формы

  • Используйте настоящие формы с тегом form, которые могут быть отправлены
  • Связывайте поля с label
  • Используйте поля ввода с правильным типом
  • Добавляйте обязательным полям атрибуты required
  • Корректно выводите сообщения об ошибках (в том числе валидации)
  • Группируйте поля ввода

Формы

        <label>

    </fieldset>

</form>

Цвет

  • Цвет не должен быть единственным источником информации, дублиуйте иконками и текстом
  • Цвета должны быть контрастными, текст - читаемым

Советы по использованию сайта

  • Важные действия стоит явно обозначать (и даже предупреждать диалогом)
  • Не должно быть временных ограничений
  • Не должно быть сложного поведения для совершения каких-то действий (доступная капча, жесты)

Порядок элементов

Порядок элементов

tabindex

  • tabindex = 0
  • tabindex = -1
  • tabindex > 0

tabindex = 0

Добавляет возможность фокусировки произвольному элементу

<button></button>
<div tabindex="0"></div>
<button></button>

tabindex = -1

Убирает элемент из обхода фокуса

<button></button>
<button tabindex="-1"></button>
<button></button>

tabindex > 0

Меняет порядок фокуса

<button></button>
<button tabindex="2"></button>
<button></button>

Focus ring

Focus ring - плохо


input {
    outline: none;
}

Focus ring - хорошо


input:focus {
    outline: 2px solid #fc0;
}
input:focus {
    outline: none;
    box-shadow 0 0 0 2px #fc0;
}

Состояние, когда нужно показать фокус

:focus матчит элемент с фокусом в любое время, но браузер показывает фокус на элементах не всегда

:focus-visible

  • Матчит элемент, если браузер считает, что нужно показать фокус
  • Chrome 67+ (Experimental Platform Features)
  • Есть полифил

:focus-visible

button {
    outline: none;
}

button:focus-visible {
    outline: 2px solid #fc0;
}

:focus-within

  • Матчит элемент, если внутри есть элемент с фокусом
  • Chrome 60+, Firefox 52+, Safari 10.1+

:focus-within

.container:focus-within {
    background: #4eb0da;
}

Атрибуты ARIA

  • Роли элементов
  • Свойства для описания элементов

Роли элементов

  • Роли отдельных виджетов: button, input, checkbox
  • Композитные роли: tablist, tree, menu
  • Структурные роли: img, list, tooltip
  • Роли регионов: main, navigation, contentinfo

Состояния виджетов

  • aria-label
  • aria-checked
  • aria-disabled
  • aria-valuemax

Связь с другими элементами

  • aria-labelledby
  • aria-controls
  • aria-activedescendant

“Живые” регионы (live-regions)

  • aria-live
  • aria-busy

Всему нужно своё имя

Любой несущий смысл контент должен быть озаглавлен. Используйте title, arial-label или aria-labelledby/aria-describedby для этого

Примеры кастомных элементов

<div tabindex="0" role="button" aria-disabled="true">
    Кнопка
</div>

Примеры кастомных элементов

<div role="tablist">
    <div role="tab" aria-selected="true" aria-controls="panel0">Первая</div>
    <div role="tab" aria-selected="false" aria-controls="panel1">Вторая</div>
</div>

Пример связи элементов

<button aria-expanded="true" aria-controls="spoilers">
    Показать спойлеры
</button>
<div id="spoilers"></div>

Пример связи элементов

<button aria-label="Отменить" aria-describedby="describe-close"></button>
<div id="describe-close">Отмена отмены может отменить отмену</div>

Советы и рекомендации

  • Если есть нативная возможность сделать что-то, используйте её
  • Не меняйте нативное поведение на любое другое
  • Все интерактивные элементы страницы должны быть доступны для управления с клавиатуры
  • Не скрывайте от программ экранного чтения фокусируемый контент на странице
  • Все интерактивные элементы должны иметь описание
